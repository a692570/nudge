# Stage 2 prompt — Convert being to doing

For every "being" item from Stage 1, convert it to a "doing" item. The doing is measurable and has a completion point. The being is a state with no end.

## The conversion rule

Being = a state. Doing = an action with a verifiable endpoint.

Examples:
- "Great cook" -> "make Christmas dinner for 8 people without help"
- "Fluent in Mandarin" -> "have a 10-minute conversation with a native speaker about a topic I choose"
- "Fit" -> "deadlift my bodyweight for 5 reps"
- "Confident public speaker" -> "give a 20-minute talk to 50 strangers without notes"
- "Good writer" -> "publish one essay per week for 12 weeks"

## How the agent handles it

For each being item, the agent suggests a conversion. The user accepts or edits. The agent does not finalize without user sign-off. If the user says "that's not quite right," ask "what would prove to you that you've become this?" The proof is the doing.

## What to refuse

Refuse conversions that are still states in disguise:
- "Be more confident" -> "speak up more in meetings" (still vague, no endpoint)
- "Be a better writer" -> "write more often" (no verifiable completion)

Push for: a specific action, a specific audience or context, a verifiable outcome. "Speak up in every team meeting for the next 4 weeks" is a doing. "Write one essay per week for 12 weeks and publish it on a blog with at least 100 readers" is a doing.

## Pacing

5 minutes. This is fast. The agent suggests, the user edits, move on. Don't belabor it. The steps generated in Stage 6 are where the real work happens.
